# VR 비상장치 교육관 프로젝트 브리프

## 1. 프로젝트 개요

프로젝트명은 임시로 **비상장치 교육관: 지하철관**으로 한다.

이 프로젝트는 Unity 기반 VR 기능성 교육 콘텐츠다. 사용자는 지하철 공간 안에서 비상장치의 위치, 기능, 사용법, 사용 순서를 직접 체험한다.

핵심 목표는 단순 정보 전달이 아니라, 실제 상황에서 어떤 장치를 먼저 사용해야 하는지 판단하고 직접 조작하게 만드는 것이다.

## 2. 개발 대상

### 플랫폼

* Unity
* VR
* Meta Quest 또는 PC VR 기준

### 장르

* VR 기능성 교육 콘텐츠
* 체험형 시뮬레이터
* 안전 교육 콘텐츠

### MVP 플레이타임

* 약 15~20분

## 3. 핵심 기획 의도

사용자는 일상에서 소화기, AED, 비상벨, 비상통화장치, 유도등 같은 비상장치를 자주 보지만 실제로 사용해본 경험은 거의 없다.

기존 안전교육은 포스터, 영상, 매뉴얼, 강의 중심이라 다음 문제가 있다.

* 실제 행동 경험 부족
* 공간 인식 부족
* 상황 판단 훈련 부족
* 압박 상황 대응 경험 부족

따라서 본 프로젝트는 VR 환경에서 비상장치를 직접 찾아보고, 선택하고, 조작하면서 행동 기억을 만드는 것을 목표로 한다.

## 4. 콘텐츠 구조

콘텐츠는 크게 두 가지 학습 방식으로 구성된다.

### 4.1 장치 학습 모드

비상장치 하나를 선택하고 해당 장치의 위치, 기능, 사용법을 학습한다.

기본 흐름:

1. 장치 선택
2. 해당 장치 존 점등
3. 중앙 스크린 설명 출력
4. NPC 설명
5. 플레이어가 직접 장치 조작
6. 결과 확인
7. 학습 완료 처리

### 4.2 상황 연계 학습 모드

비상 상황을 선택하고, 상황에 맞는 장치를 순서대로 사용하는 학습 모드다.

기본 흐름:

1. 상황 선택
2. 중앙 스크린에서 상황 연출
3. 관련 장치 존 점등
4. NPC가 첫 행동 안내
5. 장치 간 연결 라인 또는 순서 표시
6. 플레이어가 순서대로 장치 사용
7. 요약 출력
8. 학습 완료 처리

## 5. MVP 범위

### 포함 범위

MVP에서는 지하철관만 구현한다.

포함 요소:

* 지하철관 1개 맵
* 시작 화면
* 교육관 / 체험관 선택 화면
* 교육관만 활성화
* 체험관은 Locked 상태
* 중앙 스크린
* 로봇 NPC
* 손목 UI
* 장치 학습 시스템
* 상황 연계 시스템
* 3개 존
* 8개 비상장치

### 제외 범위

MVP에서 제외한다.

* 체험관 실제 플레이
* 멀티플레이
* 학교관, 상가관, 공항관 등 확장 맵
* 복잡한 자유 이동
* 장시간 이동 기반 맵 탐험

## 6. 전체 플레이 흐름

전체 흐름:

1. 시작 화면
2. 시작 버튼 선택
3. 지하철관 중앙 위치에 플레이어 스폰
4. 로봇 NPC 등장
5. NPC가 교육관 안내
6. 중앙 스크린에서 학습 방식 선택

   * 장치별 학습
   * 상황 연계 학습
7. 선택한 학습 진행
8. 장치 조작 수행
9. 결과 확인
10. 요약 출력
11. 종료

## 7. 시작 화면

### 구성

* 중앙 로고
* 교육관 로비 느낌의 배경
* 시작 버튼 1개

### 버튼 인터랙션

커서가 버튼 위에 올라가면 다음 효과를 준다.

* 외곽선 강조
* 약간 확대
* 선택음 출력

선택은 VR 컨트롤러 레이 포인터와 트리거 입력을 기준으로 한다.

## 8. 메뉴 선택 화면

시작 이후 사용자는 다음 메뉴를 본다.

* 교육관
* 체험관

MVP에서는 교육관만 활성화한다. 체험관은 비활성 상태로 표시하고 Locked 상태를 보여준다.

## 9. 지하철관 공간 구조

지하철관은 중앙 플랫폼 중심 구조다.

### 중앙

* 플레이어 시작 위치
* 로봇 NPC
* 중앙 스크린

### 주변

* 사고 대응 존
* 화재 대응 존
* 대피 / 탈출 존

공간은 자유 이동이 아니라 제한된 텔레포트 이동 중심으로 구성한다. VR 멀미를 줄이기 위해 장치 간 거리는 짧게 유지한다.

## 10. 존 구조

지하철관은 장치를 단순 나열하지 않는다. 상황 중심으로 3개 존을 구성한다.

### 10.1 사고 대응 존

목적:

* 사람의 생명 위기 상황 대응 학습

포함 장치:

* 비상통화장치
* AED
* 에스컬레이터 정지 버튼
* 열차 비상장치

대표 시나리오:

응급 환자:

1. 사람이 쓰러짐
2. 비상통화장치 사용
3. AED 사용

에스컬레이터 사고:

1. 끼임 사고 발생
2. 에스컬레이터 정지 버튼 사용
3. 비상통화장치 사용

열차 내부 사고:

1. 열차 내 위기 발생
2. 열차 비상통화 사용
3. 문 개방 관련 장치 사용

핵심 학습 포인트:

* 사람 발견
* 즉시 대응
* 도움 요청
* 후속 조치

### 10.2 화재 대응 존

목적:

* 화재 상황 대응 학습

포함 장치:

* 소화기
* 소화전
* 비상통화장치

대표 시나리오:

작은 불:

1. 작은 불 발생
2. 소화기 사용

큰 불:

1. 큰 불 발생
2. 소화전 사용
3. 비상통화장치 사용

핵심 학습 포인트:

* 불 크기에 따른 장치 선택
* 초기 화재와 확대 화재 대응 구분
* 실패 시 대피 행동으로 전환

### 10.3 대피 / 탈출 존

목적:

* 위험 상황에서 생존을 위한 이동 판단 학습

포함 장치:

* 유도등
* 비상개폐장치
* 열차 문 개방 장치

대표 시나리오:

연기 발생:

1. 연기 발생
2. 유도등 확인
3. 유도등 방향으로 이동
4. 문이 막힌 상황 발생
5. 비상개폐장치 사용

핵심 학습 포인트:

* 어디로 가야 하는지 판단
* 유도등을 따라 이동
* 막힌 출입구 대응
* 탈출 장치 사용

| AED          | 심정지 대응       |

| 유도등          | 대피 방향 안내     |

| 열차 비상장치      | 열차 내부 비상 대응  |

## 12. 주요 시스템

### 12.1 공간 탐색 시스템

사용자가 VR 공간 안에서 장치 존을 확인하고 이동할 수 있게 한다.

요구사항:

* 자유 이동 최소화
* 짧은 텔레포트 이동 사용
* 한 번에 모든 존이 과도하게 보이지 않도록 구성
* 선택된 존은 조명 또는 강조 효과 적용

### 12.2 장치 인터랙션 시스템

각 장치는 직접 조작해야 학습 완료 처리된다.

가능한 조작:

* 버튼 누르기
* 레버 당기기
* 장치 커버 열기
* 장치 집기
* 장치 사용
* 연결 확인

규칙:

1. 장치는 설명만 보는 것으로 완료되지 않는다.
2. 플레이어가 직접 조작해야 한다.
3. 잘못된 장치를 선택하면 NPC가 재안내한다.
4. 장치 사용 후 결과를 확인해야 다음 단계로 진행된다.

### 12.3 상황 이벤트 시스템

상황 연계 학습에서 비상 상황을 발생시킨다.

예시 상황:

* 응급 환자 발생
* 화재 발생
* 연기 발생
* 에스컬레이터 사고
* 열차 내부 사고

상황 발생 시 관련 장치 존을 점등한다.

### 12.4 판단 시스템

플레이어가 상황에 맞는 장치를 선택했는지 판단한다.

판단 기준:

* 올바른 장치 선택 여부
* 올바른 순서 수행 여부
* 조작 완료 여부

잘못된 선택 시 즉시 실패 처리하기보다는 NPC가 다시 안내한다.

### 12.5 피드백 시스템

플레이어 행동 결과를 제공한다.

피드백 종류:

* 성공
* 실패
* 개선점
* 다음 행동 안내

출력 위치:

* 중앙 스크린
* 손목 UI
* NPC 대사

### 12.6 상황 연계 시스템

상황 선택 시 장치 간 사용 순서를 시각적으로 보여준다.

표현 방식:

* 관련 장치 점등
* 장치 간 연결 라인 표시
* 순서 번호 표시
* 현재 수행해야 할 장치 강조

예시:

응급 상황:

1. 비상통화장치
2. AED

화재 상황:

1. 소화기 또는 소화전
2. 비상통화장치

탈출 상황:

1. 유도등
2. 비상개폐장치

## 13. VR 컨트롤러 입력 체계

기본 입력은 다음 구조를 따른다.

| 입력       | 기능              | 사용 상황               |
| -------- | --------------- | ------------------- |
| 왼쪽 썸스틱   | 스냅 회전           | 시야 방향 조정            |
| 오른쪽 썸스틱  | 짧은 텔레포트 이동      | 장치 존 이동             |
| 오른손 트리거  | UI 선택 / 오브젝트 클릭 | 메뉴 선택, 장치 선택        |
| 양손 그립 버튼 | 잡기 / 조작         | 소화기, AED, 레버, 커버 조작 |
| A / X 버튼 | 확인 / 다음 단계      | 튜토리얼 진행             |
| B / Y 버튼 | 메뉴 열기 / 뒤로가기    | 학습 메뉴, 이전 단계        |

## 14. VR UX 원칙

### 14.1 직관적 조작

VR 초보자도 버튼 두세 개만으로 진행할 수 있게 한다.

기본 조작 원칙:

* 트리거 = 선택
* 그립 = 잡기 / 조작
* 오른쪽 스틱 = 텔레포트 이동
* 왼쪽 스틱 = 스냅 회전

### 14.2 멀미 최소화

다음 방식을 사용한다.

* 자유 이동 최소화
* 텔레포트 이동 중심
* 스냅 회전 사용
* 장치 간 거리 짧게 배치
* 한 존당 3~5개 오브젝트 이하 유지

### 14.3 행동 중심 학습

텍스트 설명보다 직접 조작을 우선한다.

학습 완료 조건은 “설명 확인”이 아니라 “장치 조작 성공”이다.

## 15. UI 구조

UI는 중앙 스크린과 손목 UI로 분리한다.

### 15.1 중앙 스크린

역할:

* 상황 설명
* 장치 기능 설명
* 학습 단계 안내
* 상황 연출 표시
* 결과 요약 표시

예시 출력:

비상통화장치 학습:

* 위치: 지하철 승강장 벽면
* 사용 상황: 응급 상황 발생 시
* 기능: 역무원에게 즉시 상황 전달
* 사용 방법: 버튼을 눌러 연결

### 15.2 손목 UI

역할:

* 개인 진행 정보 표시
* 현재 단계 표시
* 다음 행동 안내
* 뒤로가기
* 설명 다시보기

예시 출력:

현재 단계:

* 비상통화장치 사용

다음 행동:

* 버튼을 눌러 연결을 시도하세요.

## 16. NPC 시스템

로봇 NPC는 튜토리얼과 상황 안내를 담당한다.

역할:

* 교육관 소개
* 장치 설명
* 상황 안내
* 잘못된 선택 시 재안내
* 학습 완료 안내

초기 대사 예시:

“안녕하세요. 이곳은 지하철 비상장치 교육관입니다.”

상황 안내 예시:

“승객이 갑자기 쓰러졌습니다. 먼저 비상통화장치로 도움을 요청하세요.”

잘못된 선택 안내 예시:

“지금 상황에서는 이 장치보다 먼저 도움을 요청하는 장치가 필요합니다.”

## 17. 학습 완료 조건

### 장치 학습 완료 조건

장치별로 다음 조건을 만족해야 완료 처리한다.

1. 장치 선택
2. 설명 확인
3. 올바른 조작 수행
4. 결과 확인

### 상황 연계 학습 완료 조건

상황별로 다음 조건을 만족해야 완료 처리한다.

1. 상황 선택
2. 첫 번째 장치 선택
3. 첫 번째 장치 조작 완료
4. 다음 장치 선택
5. 다음 장치 조작 완료
6. 전체 순서 요약 확인

## 18. 레벨 디자인 가이드

### 배치 기준

* 플레이어 기준 180도 내 핵심 정보 배치
* 이동 최소화
* 장치 간 거리 짧게 유지
* 한 존에 3~5개 오브젝트 배치
* 과밀 배치 금지

### 시야 구조

* 모든 존이 한 번에 완전히 보이지 않게 한다.
* 선택한 존은 조명으로 강조한다.
* 관련 없는 존은 어둡게 처리한다.
* 상황 연계 시 장치 순서를 빛, 라인, 번호로 표시한다.

## 22. Codex 작업 지침

Codex는 구현 시 다음 원칙을 따른다.

1. 기획 문서의 목적은 “비상장치 행동 학습”이다.
2. 단순 UI 클릭형 교육이 아니라 VR 컨트롤러 조작 기반으로 구현한다.
3. 이동은 자유 이동보다 텔레포트 이동을 우선한다.
4. VR 멀미를 줄이기 위해 공간 이동 거리를 최소화한다.
5. 장치는 설명 확인만으로 완료 처리하지 않는다. 반드시 직접 조작을 요구한다.
6. 상황 연계 학습에서는 장치 사용 순서가 중요하다.
7. 잘못된 장치 선택은 즉시 게임오버가 아니라 NPC 재안내로 처리한다.
8. MVP에서는 지하철관만 구현한다.
9. 체험관, 멀티플레이, 확장 맵은 구현하지 않는다.
10. 데이터는 하드코딩보다 장치/상황 데이터로 분리한다.

## 23. 개발 시 확인해야 할 미정 사항

다음 항목은 아직 문서에서 명확하지 않다.

* 실제 VR SDK: OpenXR, XR Interaction Toolkit, Meta XR SDK 중 무엇을 사용할지
* 목표 HMD: Meta Quest 단독 실행인지 PC VR인지
* 그래픽 스타일: 현실풍, 저폴리, 교육용 심플 스타일 중 무엇인지
* NPC 음성 사용 여부
* 음성 안내가 TTS인지 녹음 파일인지
* 실제 장치 조작의 정확도 수준
* 평가 점수 시스템 도입 여부
* 학습 완료 후 결과 리포트 저장 여부
* 시작 화면 이후 씬 분리 방식
* UI가 월드 스페이스 Canvas인지 전용 XR UI인지

## 24. 현재 기준 최종 요약

이 프로젝트는 지하철 공간에서 비상장치를 단순히 외우게 하는 콘텐츠가 아니다.

사용자는 비상 상황을 인식하고, 올바른 장치를 선택하고, VR 컨트롤러로 직접 조작하면서 대응 순서를 학습한다.

MVP는 지하철관 1개, 사고/화재/대피 3개 존, 8개 장치, 장치 학습 시스템, 상황 연계 시스템으로 구성한다.

개발의 핵심은 “상황 → 장치 선택 → 직접 조작 → 결과 확인 → 행동 기억” 흐름을 Unity VR 환경에서 안정적으로 구현하는 것이다.
